# MyFeed - Product Requirements Document (PRD)

## 📋 개요

### 제품명
**MyFeed** - 나만의 피드 큐레이션 앱

### 비전
"알고리즘이 아닌, 내가 선택한 콘텐츠만 시간순으로"

### 문제 정의
1. SNS 알고리즘이 사용자 의도와 무관한 콘텐츠를 푸시
2. 중요한 소식을 놓치거나 원치 않는 콘텐츠에 노출
3. 여러 사이트를 돌아다니며 정보를 확인하는 불편함
4. 뉴스레터가 이메일에 묻혀 읽지 못하는 상황

### 솔루션
사용자가 **직접 선택한 소스**만 **시간순**으로 보여주는 통합 피드 앱

---

## 🎯 목표 사용자 (Target Users)

### 1차 타겟: 정보 주도적 사용자
- 기술 블로그, 뉴스를 꾸준히 읽는 개발자
- 특정 주제에 대해 깊이 있는 정보를 원하는 전문가
- RSS/뉴스레터를 이미 사용하지만 더 나은 도구를 원하는 사람

### 2차 타겟: 정보 과부하를 겪는 사용자
- SNS 알고리즘에 지친 사용자
- 원하는 정보만 효율적으로 소비하고 싶은 직장인

---

## 📱 핵심 기능 (Core Features)

### MVP (v1.0)

#### 1. RSS 피드 구독 관리
- RSS/Atom 피드 URL 추가
- 피드 유효성 검증 (URL 검사 + 파싱 테스트)
- 피드 활성화/비활성화
- 카테고리별 분류
- 추천 피드 목록 제공

#### 2. 통합 피드 보기
- **시간순 정렬** (최신순)
- 소스별 필터링
- 무한 스크롤 또는 페이지네이션
- 읽음/안읽음 표시
- 썸네일 이미지 표시

#### 3. 키워드 필터링
- 보기 싫은 키워드 차단
- 숨기기 / 흐리게 표시 옵션
- 소스 단위 뮤트
- 작성자 단위 뮤트

#### 4. 북마크 (나중에 읽기)
- 원클릭 북마크
- 북마크 목록
- 읽음 표시
- 메모 추가 (선택)

#### 5. 외부 링크 열기
- 인앱 브라우저 또는 외부 브라우저
- 공유 기능

### Post-MVP (v1.5+)

#### 6. 뉴스레터 이메일 파싱
- 전용 이메일 주소 발급 (xxx@myfeed.app)
- 이메일 → 피드 아이템 변환
- 뉴스레터 소스 자동 감지

#### 7. 일간 브리핑 (텔레그램)
- 매일 아침 지정 시간에 요약 전송
- 지난 24시간 인기 글 (북마크 기준)
- 안읽은 글 개수 알림

#### 8. 고급 기능
- 오프라인 읽기
- 검색 기능
- 태그 시스템
- 다크모드
- 동기화 (여러 기기)

## 🛠 기술 스택

### 클라이언트
- **Flutter** - 크로스플랫폼 모바일 앱
- **Provider** - 상태 관리
- **Shared Preferences** - 로컬 저장 (MVP)

### 백엔드
- **Supabase** - 인증, 데이터베이스, 실시간 동기화
- **PostgreSQL** - 메인 데이터베이스
- **Supabase Edge Functions** - 서버리스 함수

### 외부 서비스
- **Telegram Bot API** - 일간 브리핑
- **RSS/Atom Parser** - webfeed_plus

## 📊 데이터 모델

### feed_sources
| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| user_id | UUID | FK → users |
| name | VARCHAR | 피드 이름 |
| url | TEXT | RSS URL |
| type | ENUM | rss/newsletter |
| icon_url | TEXT | 아이콘 URL |
| category | VARCHAR | 카테고리 |
| is_active | BOOLEAN | 활성화 여부 |
| created_at | TIMESTAMP | 생성일 |
| last_fetched_at | TIMESTAMP | 마지막 조회일 |

### feed_items
| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| source_id | UUID | FK → feed_sources |
| title | TEXT | 제목 |
| description | TEXT | 요약 |
| content | TEXT | 본문 |
| url | TEXT | 원본 링크 |
| image_url | TEXT | 썸네일 |
| author | VARCHAR | 작성자 |
| published_at | TIMESTAMP | 발행일 |
| is_read | BOOLEAN | 읽음 여부 |

### bookmarks
| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| user_id | UUID | FK → users |
| feed_item_id | UUID | FK → feed_items |
| note | TEXT | 메모 |
| tags | TEXT[] | 태그 배열 |
| created_at | TIMESTAMP | 저장일 |
| is_read | BOOLEAN | 읽음 여부 |

### filters
| 필드 | 타입 | 설명 |
|------|------|------|
| id | UUID | PK |
| user_id | UUID | FK → users |
| type | ENUM | keyword/source/author |
| value | TEXT | 필터 값 |
| action | ENUM | hide/mute |
| is_active | BOOLEAN | 활성화 여부 |

## 📐 UI/UX 설계

### 네비게이션 구조
```
Bottom Navigation:
├── 피드 (Home) - 통합 타임라인
├── 소스 (Sources) - 구독 관리
├── 저장 (Saved) - 북마크
└── 설정 (Settings) - 필터, 알림 등
```

### 핵심 화면

#### 1. 피드 화면
- 상단: 소스 필터 칩
- 카드 리스트 (썸네일, 제목, 소스, 시간)
- Pull-to-refresh
- FAB: 빠른 추가 (선택)

#### 2. 소스 관리 화면
- 카테고리별 그룹
- 스와이프 삭제
- 토글 활성화
- 상세 정보 (마지막 업데이트 등)

#### 3. 북마크 화면
- 저장된 글 목록
- 읽음/안읽음 필터
- 태그 필터

#### 4. 설정 화면
- 필터 관리
- 알림 설정
- 텔레그램 연동
- 테마 설정

## 📈 성공 지표 (Metrics)

### 핵심 지표
1. **DAU/MAU** - 일간/월간 활성 사용자
2. **피드 소스 수** - 사용자당 평균 구독 수
3. **읽은 글 수** - 사용자당 일간 소비량
4. **북마크율** - 저장하는 글의 비율

### 품질 지표
1. **피드 파싱 성공률** - 95% 이상 목표
2. **앱 크래시율** - 0.1% 미만 목표
3. **피드 로딩 시간** - 2초 미만
